============================= модули sass =============================

- есть 6 основных модулей:
  - reset - сброс и нормализация дефолтных стилей
  - assets - стили внешних библиотек/плагинов
  - common - общие стили (корневые переменные, шрифты, иконки, базовые стили)
  - blocks - стили блоков
  - containers - стили контейнеров
  - pages - стили страниц
- каждый модуль представляет папку из файлов-подмодулей и index.scss,
  который @forward все эти подмодули

============================= подмодули sass =============================

- подмодуль - файл, содержащий 1 публичный миксин с таким же названием, как и у файла
  и какое-то кол-во приватных миксинов
- у всех подмодулей внутри модуля должны быть разные публичные миксины, т.к. они @forward
  в index.scss и могут перезаписать друг друга
- публичный миксин может быть c методами или без методов:

  @mixin normalize {
    ...
  }

  @mixin b-logo($action, $args...) {
    .b-logo {
      @if $action == 'init' { @include _main; }
      @else if $action == 'area' { @include area($args...); }

      @else { @error 'No such action - #{$action}'; }
    }
  }
- псеводоклассы и модификаторы настриваются через css переменные в приватных миксинах 

============================= структура взаимодействия модулей sass =============================

- взаимодействие с модулями осуществляется через публичный миксин:

  @include reset.normalize;
  @include blocks.b-logo('area', l, start, center);

- index - инклудит страницы
  - pages - инклудит основные модули, настраивает контейнеры под медиа запросы
    - reset
    - assets
    - common
    - containers - инклудит и настраивает блоки
      - blocks
